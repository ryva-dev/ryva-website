# Journey 5: Create a Placement Opportunity

**Trigger:** Representative identifies a qualified Product–Brand–Business match with authority and a next action.

**Required records:** active Agreement/Territory, Brand, Product(s), Business, fit rationale/match thesis, owner, Evidence, Triangle Review, next action; Contact may be pending if decision path is documented.

## Flow

1. User selects Agreement, Product(s), Business, and context.
2. System validates authority, dates, territory/channel, duplicates, account/protection conflicts, Product/Brand conditions.
3. User writes or confirms match thesis and Buyer-value rationale.
4. AI may suggest gaps, fit factors, risks, and preparation actions.
5. User completes Relationship Triangle and evidence confidence.
6. System creates Identified Opportunity with owner and next action.
7. User completes qualification gates to move to Qualified.

**Automation:** duplicate/conflict check, required evidence, initial task, stalled timer.

**Approvals:** Human match thesis, Triangle condition, and Opportunity creation.

**Success:** one connected Opportunity that meets definition and enters Identified/Qualified honestly.

**Failure:** no authority, poor fit, blocking risk/conflict, missing Product/Business, duplicate, one-sided Triangle.

**Recovery:** create research task instead, resolve authority/conflict, revise scope, or disqualify.

**Audit events:** validation, conflict result, Triangle, AI disposition, creation, stage, owner, estimates.
