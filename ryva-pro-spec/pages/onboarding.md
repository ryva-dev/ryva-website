# Page: Onboarding

## Purpose and user

Create a usable professional workspace through actual setup after certification and subscription. Primary user is a newly entitled Representative.

## Data displayed

Access status; step list; profile; experience; categories/geography; existing relationships; communication settings; imports; first Product, Brand, Business, Opportunity, and outreach readiness.

## Actions

Primary changes by step: Save and continue, verify/import, create first record, complete setup.  
Secondary: Save and exit, skip optional step, use sample explanation, connect later.

## Filters

Import steps support file/record type and error filter.

## States

- **Empty:** each step explains why the field matters and minimum input.
- **Loading:** per-step save/import status without blocking completed sections.
- **Error:** inline validation; provider/import errors preserve work and recovery.

## Permissions and responsive

Active entitled Representative. Read-only/grace users may view prior setup but not onboard. Mobile supports manual setup; bulk import is desktop-first.

## Linked records and AI

Creates Profile, preferences, Products, Brands, Businesses, Contacts, Representation/Placement Opportunities, Tasks, and integrations. AI may suggest categories, normalized fields, evidence gaps, and drafts; user confirms every material value.

## Acceptance criteria

- progress resumes accurately;
- no Opportunity/outreach bypasses authority;
- imports preserve origin;
- setup may complete without sending communication;
- Home receives a valid next action;
- every created record is linked and auditable.
