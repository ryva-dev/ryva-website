# Brand Placement Lifecycle

- **Purpose:** Orient a representative across the end-to-end professional lifecycle without turning it into a score or game.
- **Used on:** Home orientation, Product/Brand/Placement relationship overview, optional training/help context.
- **Nodes:** Discover Product → Evaluate Product + Brand → Qualify → Secure Representation → Target Business → Prepare → Approved approach → Present/Buyer review → Order discussion → Verified opening Order → Support active Account → Reorder review → Grow/Maintain/End.
- **Completed state:** Current valid evidence and required human decision satisfy the stage; completed date available.
- **Current state:** The record’s real current lifecycle stage; one stage only.
- **Blocked state:** Missing evidence, failed fit/triangle, conflict, invalid authority, unsupported claim, opt-out, or specialist stop; exact reason displayed.
- **Required next action:** Existing owned Task or next action; missing ownership is itself a blocker.
- **Backflow:** Changed evidence may return to Evaluate/Qualify; terms failure may return to Buyer review; reason is recorded.
- **Visual form:** Compact horizontal path on wide desktop, not a freeform canvas. Only adjacent stages and current dependencies are expanded.
- **Accessibility alternative:** Ordered list/table with stage, state, completed date, blocker, owner, next action.
- **Mobile alternative:** Vertical step list centered on current stage; completed stages collapsed; blocker and next action expanded.
- **Acceptance criteria:** No stage changes from the visualization without existing review; no scores/rewards; server stage and audit remain authoritative.
