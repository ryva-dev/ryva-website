# Certification and Access Progress

- **Purpose:** Explain credential and subscription dependencies for access and remediation without gamification.
- **Used on:** Access, Certification, Subscription/Profile menu.
- **Nodes:** Identity verified → Eligible certification verified → Credential term active/expiring/grace/etc. → Subscription entitled/retry/etc. → Terms/security current → Access mode determined.
- **Completed state:** Current trusted source confirms the requirement.
- **Current state:** Access mode evaluation or active renewal/activation step.
- **Blocked state:** Expired beyond grace, suspended, revoked, ineligible credential, failed entitlement, security/terms requirement, legal hold where applicable.
- **Required next action:** Renew/verify credential, manage subscription, complete security/terms, contact support/appeal.
- **Visual form:** Compact dependency stepper; no percentage, badge, celebration, course language, or “level.”
- **Accessibility alternative:** Requirements list with state, source/verified date, effect, remediation.
- **Mobile alternative:** Vertical dependency list with one remediation action.
- **Acceptance criteria:** Credential always outranks subscription, grace/access matrices remain exact, renewal does not execute overdue actions, and status is externally trusted/audited.
