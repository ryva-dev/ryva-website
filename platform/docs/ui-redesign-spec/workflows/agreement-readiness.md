# Agreement Readiness

- **Purpose:** Make the prerequisites for valid representation authority visible before approval.
- **Used on:** Agreement detail and authority ContextRail.
- **Nodes:** Immutable original received → Malware scan clean → Correct Brand/parties identified → Material terms extracted → Material terms human reviewed → Product/territory/channel scope reviewed → Commission/order/reorder/protection/termination rights reviewed → Conflicts/ambiguity resolved or flagged → Human approval → Active within dates.
- **Completed state:** Exact artifact/version and human reviewer are recorded for the requirement.
- **Current state:** First unresolved required review, without implying sequential legal interpretation.
- **Blocked state:** Scan failed/quarantine, missing artifact, unresolved conflict, legal ambiguity, missing material term, expired/effective-date failure, missing human approval.
- **Required next action:** Review named term, obtain clarification/specialist review, resolve conflict, or issue human disposition.
- **Visual form:** Dependency checklist with connectors only where one requirement truly depends on another; not an automatic progress percentage.
- **Accessibility alternative:** Requirements table with evidence link, reviewer, reviewed date, state, blocker.
- **Mobile alternative:** Readiness summary + expandable requirements; exact document full-screen.
- **Acceptance criteria:** AI-extracted fields stay labelled/editable/evidence-linked; system never interprets ambiguity or approves; no percentage score.
