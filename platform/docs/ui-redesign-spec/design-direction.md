# Design Direction

## North star

Every primary screen must quickly answer:

1. What changed?
2. What requires attention?
3. What should happen next?
4. Which Product, Brand, Buyer, authority, Placement, Account, Order, and Commission records connect?
5. What evidence supports the current state?

The intended reaction is: **“This is serious professional software built specifically for Brand Placement.”**

## Inspiration allocation

### Venture foundation — 60%

Use the reference principles for:

- modern neutral typography;
- a restrained persistent sidebar;
- compact page headers;
- generous but purposeful whitespace;
- high-quality tables and analytical layouts;
- low-chroma surfaces;
- minimal shadows;
- disciplined density.

Do not copy its exact logo, layout, component styling, chart design, or content.

### Relationship-centered depth — 30%

Use the reference principles for:

- record identity headers;
- a central operational or communication timeline;
- actions adjacent to the relevant relationship;
- a contextual right rail for next action, risk, authority, evidence gaps, and commercial context;
- split views that retain list context while reviewing a record.

Do not copy bright lime, playful bubbles, social-profile styling, circular action clusters, or chat-first composition.

### Stratus workflow thinking — 10%

Use the reference principles for:

- stage dependencies;
- completed/current/blocked/available states;
- visibly connected prerequisites;
- a clear required next action.

Use only for the seven workflows listed in `workflows/README.md`. Do not create freeform process canvases or represent ordinary tables as node diagrams.

## Product character

| Attribute | Expression |
|---|---|
| Premium | Precise typography, measured spacing, quiet surfaces, careful details |
| Modern | Efficient split views, responsive drawers, command search, predictable state feedback |
| Calm | Low-chroma palette, limited accent, one primary action, no decorative motion |
| Analytical | Comparable tables, tabular numbers, explicit definitions, freshness and provenance |
| Disciplined | Stable anatomy, limited variants, consistent language, clear action hierarchy |
| Trustworthy | Evidence adjacent to claims, unknowns visible, permissions and human ownership explicit |
| Brand Placement-specific | Relationship triangle, authority, Placement stage, Buyer fit, account continuity, commissions |

## Composition rules

1. Information precedes decoration.
2. One primary action is visually dominant per page or review state.
3. Tables are the default for comparable records; cards are reserved for emphasis, Kanban, or concise summaries.
4. The page canvas is open; borders define data or interaction boundaries, not every section.
5. Context remains visible through an identity header, breadcrumb/relation trail, selected list item, or right rail.
6. Evidence, risk, authority, and freshness appear next to the decision they govern.
7. Historical detail is progressively disclosed through tabs, drawers, or timelines.
8. AI appears as a labelled recommendation or review artifact, never as a floating chatbot.
9. Status labels are compact and restrained. Color is supplemental.
10. Human approval receives more spacing and explicit language than ordinary edits.

## Anti-patterns

Reject a design if it introduces:

- equal-weight dashboard card grids;
- nested cards or excessive rounded containers;
- bright status colors across the page;
- giant internal hero headings;
- false urgency, gamification, or celebratory animation;
- deep navigation nesting;
- icon-only consequential actions;
- decorative charts or unlabelled metrics;
- content-minimizing layouts that remove professional context;
- AI chat as the primary interaction;
- copied visual treatment from any supplied reference.

## Calm density

“Spacious” does not mean low-information. Use:

- 40–48 px table rows for routine scanning;
- compact 32 px filters on desktop;
- 24–32 px section intervals;
- 64–80 px page gutters on wide screens;
- stable labels and aligned values;
- details on demand for provenance and history;
- a 320 px right rail only when its context materially affects the task.

Whitespace increases around high-consequence approval, blocking reasons, and record identity. It decreases in registers, timelines, and comparison tables where adjacency improves comprehension.

## Originality boundary

The specification borrows design principles, not assets or compositions. Ryva’s distinction comes from its evidence language, human approval, authority state, relationship context, Placement lifecycle, and commercial-continuity model.
